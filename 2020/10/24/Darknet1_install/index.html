Dark net起源

在介绍原作者Joseph Redmon大神之前，先夸两位更加让人熟知的国内大神，他们的成就和经历比较相似。

第一位，贾扬清（清本硕，UCLA CS-phd），曾于新加坡国立大学、微软亚洲研究院、NEC美国实验室、Google Brain工作，现任Facebook研究科学家，负责前沿AI平台的开发以及前沿的深度学习研究。前几年火遍深度学习圈的学习框架Caffe是其在博士期间的一部分作品！（注意只是一部分哈哈哈哈艹）

第二位，李沐（上交ACM班，CMU CS-phd）,曾实习于微软亚研院、香港科技大学助理研究员、百度高级研究员和首席架构师，现任亚马逊首席科学家，因为李沐在博士期间创作了现亚马逊的深度学习框架MXnet的原型。

对于Joseph Redmon（详细简历）（University of Washington，CS），他的作品比Darknet更出名的是他提出的目标检测算法：YOLO 为了体现出大神的独一无二，他决定为Yolo打造属于自己的学习框架（这个是我杜撰的hh，应该不会错），整个框架几乎完全基于C编写，非常轻量化，当然也很小众化（原因是像我这样的调包侠不会用啊……）。

基于Darknet的优点+YOLO系列的加持，Joseph Redmon大神专门做了一个开源网站，下面是他的 paper，不过他已经一年多没有更新自己的Darknet了。现在继续维护优化Darknet的是一位俄罗斯的大神AlexayAB(不知道是不是他的名字，详细见文头链接)

Dark net优势

  • 易于安装：在makefile里面选择自己需要的附加项（cuda，cudnn，opencv等）直接make即可；

    • OpenCV if you want a wider variety of supported image types.
    • CUDA if you want GPU computation.

  • 独立性好：整个Darknet是个独立的C语言包，客观来说可以完全不用依赖外部的各种包就可以实现深度学习的任务，但也提供了opencv和cuda的接口，opencv可以实现更加多样的图片接口，cuda为了适应GPU训练，opencv作者也为Darknet编写了可以对其进行替代的函数；

  • 源码结构清晰：下面是github上源码布局:

文件夹名称 包含内容
cfg 配置整个模型的构架，也即backbone的定义（已经包含有常见的检测、分割的backbone）
data 存有一些数据的label，还有可直接训练的coco数据集，但是对于自定义的数据，于此无关
examples 高级函数（如检测函数classifier等），用户最常用的函数文件夹
include C语言头文件
python 用python调用模型，主要在darknet.py中（调用时还需用到动态库libdarknet.so）
scripts 常用的脚本（如coco数据集下载，voc数据转化为训练所需要的数据格式）
src 最顶层框架定义文件，也是darknet的源码
  • 易于移植：（1）得益于C语言，故基于Darknet的项目都会很容易部署到本地机器上，使用CPU或者GPU；（2）可以利用darknet.py很容易将训练好的模型用python调用
